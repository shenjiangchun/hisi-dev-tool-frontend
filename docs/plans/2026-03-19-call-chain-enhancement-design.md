# 调用链分析页面增强设计文档

> **日期**: 2026-03-19
> **状态**: 待实现

## 概述

增强现有的调用链分析功能，新增方法引用关系分析页面，支持通过 URI 下拉选择和节点右键菜单交互方式查询调用链。

## 需求概述

### 页面结构

```
侧边栏菜单
├── 调用链分析
│   ├── 接口调用链查询（改造现有 CallChainGraph）
│   └── 方法引用关系分析（新增页面）
```

### 功能需求

#### 1. 接口调用链查询页面（改造）

- URI 下拉选择框，支持模糊搜索
- 调用链图表展示，支持 Tree/Graph 视图切换
- 节点右键菜单：向上/向下查询调用关系
- 右键菜单支持跳转到方法引用关系分析页面

#### 2. 方法引用关系分析页面（新增）

- 类名下拉选择框，支持模糊搜索
- 方法名下拉选择框
- 查询方向选择：向上/向下
- 调用关系图表展示，支持 Tree/Graph 视图切换
- 支持独立访问（侧边栏菜单入口）

## 技术设计

### 后端 API

#### 新增接口

**1. 获取类名列表**
```
GET /api/callchain/classes?project={project}
返回: [{className: "xxx", packageName: "com.xxx", methodCount: 5}]
```

**2. 获取类的方法列表**
```
GET /api/callchain/methods?className={className}&project={project}
返回: [{methodName: "xxx", signature: "void xxx(String a)", line: 25}]
```

#### 复用现有接口

- `GET /api/callchain/uris` - 获取 URI 列表
- `GET /api/callchain/calls` - 获取 URI 调用链
- `GET /api/callchain/callers` - 获取上游调用链
- `GET /api/callchain/callees` - 获取下游调用链

### 前端组件结构

```
src/views/call-chain/
├── CallChainGraph.vue        # 改造：接口调用链查询
├── MethodReferenceGraph.vue  # 新增：方法引用关系分析
└── components/
    ├── UriSelector.vue       # URI 下拉选择组件
    ├── ClassMethodSelector.vue # 类方法选择组件
    ├── ChainChart.vue        # 图表组件（Tree/Graph视图切换）
    └── ContextMenu.vue       # 右键菜单组件
```

### 右键菜单选项

```
节点右键菜单：
├── 查看上游调用链（谁调用了此方法）
├── 查看下游调用链（此方法调用了谁）
├── 复制方法签名
└── 在方法引用分析中打开
```

```typescript

{
```

### ECharts 图表类型

- **Tree 视图**: 从左到右展示调用层级，适合展示树形结构
- **Graph 视图**: 节点用线连接，支持缩放、拖拽、高亮，适合展示复杂关系
